あなたの役割は、この`{{技術要件定義書}}`を**正確かつ徹底的に参照**し、それに準拠した包括的な開発計画ファイル`{{develop_plan.yaml}}`を生成することである。この計画書は、`{{技術要件定義書}}`で示されたアーキテクチャや開発要件に応じて、適切な開発フェーズ、ステップ、実行コマンド、テスト戦略などを反映したものとする。
トークンの限界まで正確かつ詳細に記述してください（技術要件定義書のどこを確認すればいいかも含め）。サボって抜け漏れがあって実装が間違った場合実装者が逮捕される可能性があります。

#### 要求事項

1. **出力形式**:  
   YAML形式で出力し、トップレベルキーとして`phases`を含む構造とする。  
   ```yaml
   phases:
     - name: "..."
       steps:
         - "..."
         ...
のような形式を基本とする。フェーズ数や名前は{{技術要件定義書}}の内容やプロジェクト特性に応じて柔軟に決めてよい。

2. **フェーズ分割の柔軟性**:  
   開発計画は複数のフェーズ(例: 開発環境構築、バックエンド実装、フロントエンド実装、統合テスト、デプロイ、改善計画など)に分割する。ただし、これは固定ではなく、{{技術要件定義書}}のプロジェクト特性や規模、技術選択に応じて適切な段階分けを行うこと。
例えば、もしプロジェクトにNext.jsとFastAPIが含まれるなら、初期フェーズでNode, Python, Docker, CIなどの安定的な環境構築を行うフェーズを用意し、続いてバックエンドコア機能やフロントエンド機能実装フェーズへ進む。もし他の技術スタックならそれに合わせたフェーズを定義する。

3. **安定した環境構築手順の詳細化**:  
   {{技術要件定義書}}の内容を参照し、開発者が誰でも同じ手順で構築可能なように、環境構築手順を極めて詳細に記述すること。
具体的には以下を考慮するが、プロジェクトに不要な場合は省いてよい。

 - バージョン確認（node -v, python --version, docker --version, docker-compose --version）
 - OSによる差異（Windows/Mac/Linux）の注意点、WSL2の設定例Node.jsやPythonのバージョン管理手法（nvm, pyenvなど）、仮想環境構築（python -m venv venvなど）
 - Next.jsなどのフロントエンドフレームワークの初期化コマンド例（npx create-next-app@latestなど）、もしこのプロジェクトがReact, Vue, Angularならそれぞれの標準的初期化コマンドを提案する
 - Pythonパッケージインストール（pip install ... && pip freeze > requirements.txt など）、バックエンドサーバー起動（uvicorn app.main:app --reload）手順
 - Dockerビルド・Docker Compose起動（docker-compose up --build）手順と、起動後の稼働確認例
 - GitHub ActionsなどのCI/CDでの環境セットアップ（actions/setup-node, actions/setup-pythonなど）とテスト実行手順
開発計画はプロジェクトに特有のディレクトリやファイル構成（{{技術要件定義書}}で定義）を踏まえ、それらを参照したり配置したりする手順を明示すること。

4. ** テスト戦略の明確化**:  
 - フェーズごとに、シェルコマンドでテストを実行する方法をtest_phase(1,2,3...).mdとして記述する。
 - 実行するシェルコマンドをtest_phase(1,2,3...).mdにちゃんと記載すること

5. **最終的な出力**:  
   最終的な出力は、phasesキーをトップに置いたYAML形式で、nameとstepsを持つフェーズが複数含まれ、各ステップが詳細な作業ガイドを記した{{develop_plan.yaml}}を出力すること。
   固定的なフェーズ数やステップ数、技術セットアップ手順に依存しすぎず、{{技術要件定義書}}内のアーキテクチャと要件、利用技術に基づいて柔軟に決定すること。
